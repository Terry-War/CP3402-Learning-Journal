# Week 04 - learning Journal 
## Learning Activities & Resources
Set up and tested a local development environment for a WordPress website using **Local by Flywheel**. Working locally made the development process significantly faster compared to using CloudAccess hosting, which previously had a 4–5 second delay when saving changes or updating pages. The faster feedback loop allowed me to experiment more with layouts and content without waiting for the server to refresh each time.

During this process I also reviewed WordPress templates and discovered that the **Astra template is not permitted for the assessment**, meaning I will need to select a different theme that still provides good layout flexibility and accessibility. My plan is to continue expanding this locally developed version of the site for the main WordPress assessment and then convert the design and content into a **Joomla version** for the second part of the assignment due in two weeks.

## Content Insights

### Career
Learning how to develop websites locally is an important workflow skill used by professional web developers. Many developers build and test sites locally before deploying them to a live server to avoid performance issues and accidental errors on production environments. Understanding both local development and cloud hosting workflows is valuable for roles in web development, CMS administration, and digital project management.

### Employability
Using tools such as Local by Flywheel demonstrates familiarity with common development practices such as staging environments and deployment workflows. Employers often expect developers to know how to build, test, and troubleshoot websites locally before publishing them online. This experience also highlights the importance of choosing the right development environment and tools to improve productivity and reduce delays when making changes to a website.

### Learning Insights
This activity showed how much faster and easier it is to develop a WordPress site locally compared to working directly on a hosted server. The ability to instantly see changes improved my workflow and allowed me to experiment with design and layout more efficiently. I also learned the importance of checking assessment requirements when choosing themes or plugins, as some popular options may not be permitted. In future projects I would start development locally first and then migrate the finished site to a hosting provider once the design is finalized.
